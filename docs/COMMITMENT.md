## The Hard Question: Operational Responsibility

### Who handles emergencies when triggered?

**Answer:** Start semi-manual, build tech to replace humans gradually, never pretend automation exists before it does.

### Week 1-2: Manual Operations
- **You / Operations Team:** Manual routing, assignment, coordination
- **Facilities:** Manual acknowledgment (phone or app)
- **Drivers:** Manual status updates (phone or app)
- **Goal:** Prove the loop works, even if manual

### Week 3-4: Semi-Automated
- **System:** Automatic routing to nearest facility
- **You:** Manual driver assignment (if needed)
- **Facilities:** App-based acknowledgment
- **Drivers:** App-based status updates
- **Goal:** Reduce manual steps, maintain reliability

### Post-Week 4: Fully Automated (Future)
- **System:** Full automation with human oversight
- **Operations Team:** Monitor dashboard, intervene only when needed
- **Goal:** Reliability with efficiency

**Key Principle:** Zambia rewards reliability, not sophistication.

## Risk Mitigation

### Risk: 4-week deadline too aggressive
**Mitigation:** Focus on closed loop, not perfection. Manual steps OK.

### Risk: No drivers available
**Mitigation:** Start with facility-only routing. Add drivers in Week 3.

### Risk: Facilities don't respond
**Mitigation:** Build acknowledgment system. Track response rates.

### Risk: Operational burden
**Mitigation:** Start manual, automate gradually. Never fake automation.
